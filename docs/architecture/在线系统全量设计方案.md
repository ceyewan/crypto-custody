# 虚拟货币存管提控原型系统在线端全量设计方案

## 1. 文档目标

本文档描述虚拟货币存管提控原型系统在线端的目标设计。在线端设计以覆盖测试大纲为边界，不额外扩大为密钥管理系统或安全芯片管理系统。

在线端负责案件、用户、账户、交易、审计、备份和测试数据管理；离线端负责私钥、安全芯片、警员参与方、门限密钥生成、门限签名、私钥移交和私钥销毁。

本文档用于指导后续 `online-server` 和 `online-client` 的代码改造。

## 2. 在线端职责边界

### 2.1 在线系统负责

- 用户管理：管理员、操作员、审计员等角色的登录、认证、权限和增删查改。
- 案件管理：创建案件、维护案件信息、查询案件、关联涉案账户和案件托管钱包。
- 账户管理：导入收缴原始账户、记录离线系统生成的案件托管钱包地址、查询余额、批量导入和分页查询。
- 交易管理：构建交易、生成待签名哈希、导出签名任务、导入签名结果、广播交易、查询交易状态。
- 审计日志：记录用户、案件、账户、交易、导入导出、备份恢复等关键操作。
- 备份恢复：在线业务数据库热备份、加密冷备份、恢复和验证。
- 测试数据：生成或导入满足测试规模的数据，例如用户、案件、1000 个账户、500 笔交易。

### 2.2 在线系统不负责

- 安全芯片导入、登记、状态检测和 CPLC 管理。
- 警员和安全芯片之间的映射。
- 私钥、助记词、keystore 或密钥分片的导入和存储。
- 私钥移交、私钥销毁和分片重分发。
- 门限密钥生成过程。
- 门限签名过程。
- 分片分布查询。

这些能力全部归属离线系统。在线系统最多保存离线系统返回的业务结果，例如案件托管钱包地址、签名结果、离线任务编号或结果包哈希。

## 3. 总体主线

系统主线如下：

1. 管理员或操作员在在线系统创建案件。
2. 在线系统导入案件涉及的收缴原始账户。
3. 在线系统为案件创建“生成托管钱包”的离线任务。
4. 离线系统完成门限密钥生成，生成案件托管钱包地址。
5. 在线系统导入离线结果，保存案件托管钱包地址。
6. 如需归集资产，在线系统构建从收缴原始账户到案件托管钱包的交易，并生成待签名哈希。
7. 离线系统对交易哈希签名。
8. 在线系统导入签名结果并广播交易。
9. 后续提取案件托管钱包资产时，仍由在线系统发起交易、离线系统签名、在线系统广播。
10. 全流程记录审计日志。

在线系统和离线系统的交互通过任务包和结果包完成，不做实时同步。

## 4. 角色权限设计

| 角色 | 定位 | 在线端权限 |
| --- | --- | --- |
| admin | 系统管理员 | 用户管理、案件管理、账户管理、交易管理、备份恢复、测试数据、审计查询 |
| officer | 操作员 | 案件维护、账户导入、余额查询、交易申请、任务包导出、签名结果导入、交易广播 |
| auditor | 审计员 | 用户、案件、账户、交易和审计日志只读查询，审计导出 |

权限要求：

- 所有业务写操作必须登录。
- 用户管理、备份恢复、测试数据生成只允许管理员操作。
- 案件、账户和交易操作允许管理员和操作员操作。
- 审计员只读。
- 关键操作成功和失败都必须记录审计日志。

## 5. 核心数据模型

### 5.1 User 用户

保存在线系统用户和角色。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| username | 用户名，唯一 |
| password | 密码哈希 |
| email | 邮箱，唯一 |
| role | admin/officer/auditor |
| status | active/disabled |
| last_login_at | 最近登录时间 |
| created_at / updated_at / deleted_at | 通用时间字段 |

### 5.2 Case 案件

保存案件主数据。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| case_no | 案件编号，唯一 |
| name | 案件名称 |
| description | 案件描述 |
| status | active/closed/archived |
| custody_account_id | 当前案件托管钱包账户 ID |
| custody_address | 当前案件托管钱包地址 |
| created_by | 创建人 |
| created_at / updated_at / deleted_at | 通用时间字段 |

说明：

- `custody_address` 来自离线系统生成结果。
- 在线系统只记录地址，不记录安全芯片、警员、密钥分片。

### 5.3 Account 账户

保存收缴原始账户和案件托管钱包账户。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| address | 虚拟货币地址，唯一 |
| coin_type | 币种，例如 ETH |
| account_type | seized_original/custody_wallet |
| balance | 当前余额，字符串保存 |
| balance_source | chain/manual/test |
| case_id | 关联案件 ID |
| case_no | 关联案件编号 |
| imported_by | 导入人 |
| source | manual/csv/json/offline_result/test |
| key_material_hint | none/has_private_key/offline_generated/offline_signed |
| offline_ref_no | 离线任务或结果引用编号，可选 |
| description | 备注 |
| last_balance_sync_at | 最近余额同步时间 |
| created_at / updated_at / deleted_at | 通用时间字段 |

账户类型：

- `seized_original`：收缴原始账户，来自案件侦办材料，可能只有地址，也可能线下掌握私钥。
- `custody_wallet`：案件托管钱包，由离线系统通过门限密钥生成后返回地址。

`key_material_hint` 只描述业务状态，不保存私钥：

- `none`：只有地址，没有私钥材料。
- `has_private_key`：线下掌握私钥材料，但私钥不进入在线系统。
- `offline_generated`：离线系统生成的托管钱包。
- `offline_signed`：该账户可通过离线系统完成签名。

### 5.4 Transaction 交易

保存归集交易、提取交易和模拟交易。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| tx_no | 交易编号 |
| case_id / case_no | 案件 |
| tx_type | collect/withdraw/test |
| from_account_id | 发送账户 ID |
| from_address | 发送地址 |
| to_address | 接收地址 |
| value | 金额 |
| coin_type | 币种 |
| reason | 交易事由 |
| message_hash | 待签名哈希 |
| unsigned_payload | 待签名交易数据 |
| signature | 导入的签名结果 |
| tx_hash | 链上交易哈希 |
| receipt | 链上回执 |
| status | 状态 |
| created_by | 创建人 |
| approved_by | 审批人，可选 |
| exported_at | 签名任务导出时间 |
| signed_at | 签名结果导入时间 |
| broadcasted_at | 广播时间 |
| confirmed_at | 确认时间 |
| created_at / updated_at / deleted_at | 通用时间字段 |

交易状态：

```text
draft
pending_signature
signature_exported
signed
broadcasted
confirmed
failed
cancelled
```

说明：

- 在线系统负责构建交易和生成哈希。
- 离线系统只负责签名。
- 在线系统导入签名后负责广播和状态更新。

### 5.5 OfflineTask 离线任务记录

保存在线系统导出给离线系统的任务摘要。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| task_no | 任务编号 |
| task_type | custody_keygen/sign |
| case_id / case_no | 案件 |
| account_id | 相关账户，可选 |
| transaction_id | 相关交易，可选 |
| payload_hash | 任务包哈希 |
| result_hash | 结果包哈希 |
| status | exported/imported/completed/failed |
| exported_by | 导出人 |
| imported_by | 导入人 |
| exported_at | 导出时间 |
| imported_at | 导入结果时间 |
| created_at / updated_at | 时间字段 |

说明：

- 该表只用于在线端追踪任务包和结果包。
- 不保存警员、安全芯片和分片信息。

### 5.6 AuditLog 审计日志

记录关键业务操作。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| request_id | 请求编号 |
| username | 操作用户 |
| role | 操作角色 |
| action | 操作动作 |
| resource_type | 资源类型 |
| resource_id | 资源 ID |
| case_no | 案件编号 |
| ip | 客户端 IP |
| user_agent | 浏览器标识 |
| before_data | 操作前数据摘要 |
| after_data | 操作后数据摘要 |
| result | success/failure |
| error_message | 错误信息 |
| latency_ms | 响应耗时 |
| created_at | 创建时间 |

审计日志不得记录明文私钥、密码、密钥分片、助记词等敏感材料。

### 5.7 BackupRecord 备份记录

记录热备份和冷备份。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| backup_no | 备份编号 |
| backup_type | hot/cold |
| file_name | 文件名 |
| file_path | 文件路径 |
| file_hash | 文件哈希 |
| encrypted | 是否加密 |
| created_by | 创建人 |
| restored_by | 恢复人 |
| restored_at | 恢复时间 |
| status | created/restored/failed |
| created_at / updated_at | 时间字段 |

### 5.8 Job 批量任务

支撑批量导入、批量生成和批量余额同步。

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| job_no | 任务编号 |
| type | import_accounts/seed_test_data/sync_balances/export_backup |
| status | pending/running/success/failed/partial_success |
| total | 总数 |
| success | 成功数 |
| failed | 失败数 |
| progress | 进度 |
| result_file | 结果文件 |
| created_by | 创建人 |
| started_at / finished_at | 时间字段 |
| error_message | 错误摘要 |

## 8. 关键业务流程

### 8.1 案件创建和账户导入

1. 管理员或操作员创建案件。
2. 导入收缴原始账户。
3. 选择账户类型为 `seized_original`。
4. 如线下掌握私钥，只记录 `key_material_hint = has_private_key`。
5. 在线系统保存案件和账户，并记录审计日志。

### 8.2 案件托管钱包生成

1. 在线系统为案件创建托管钱包生成任务。
2. 在线系统导出任务包。
3. 离线系统导入任务包，完成门限密钥生成。
4. 离线系统导出结果包，包含托管钱包地址。
5. 在线系统导入结果包。
6. 在线系统创建 `custody_wallet` 类型账户，并更新案件 `custody_address`。

### 8.3 资产归集交易

1. 在线系统选择收缴原始账户和案件托管钱包。
2. 在线系统查询余额、nonce、gas 和 chainId。
3. 在线系统构建交易并生成待签名哈希。
4. 在线系统导出签名任务包。
5. 离线系统签名。
6. 在线系统导入签名结果。
7. 在线系统广播交易并查询回执。

### 8.4 案件资产提取交易

1. 在线系统从案件托管钱包创建提取交易。
2. 在线系统构建交易并生成待签名哈希。
3. 在线系统导出签名任务包。
4. 离线系统使用门限签名生成签名结果。
5. 在线系统导入签名、广播交易、查询回执。
6. 全流程记录审计日志。

### 8.5 备份恢复

1. 管理员创建热备份或冷备份。
2. 冷备份文件加密并保存哈希。
3. 模拟数据丢失后，管理员上传备份文件并恢复。
4. 恢复后查询案件、账户和交易数据，验证一致性。

## 9. 性能和批量处理

为满足关键操作 3 秒响应要求：

- 案件、账户、交易、审计日志列表全部后端分页。
- 账户列表默认每页 20 或 50 条。
- 1000 条账户导入使用批量任务。
- 500 笔交易测试数据生成使用批量任务。
- 地址、案件编号、账户类型、交易状态、创建时间建立索引。
- 余额同步不要阻塞列表加载。
- 前端只渲染当前页数据。

关键响应目标：

| 操作 | 目标 |
| --- | --- |
| 登录到主界面加载 | 平均不超过 3 秒 |
| 查询案件列表第一页 | 平均不超过 3 秒 |
| 查看案件详情 | 平均不超过 3 秒 |
| 提交交易申请 | 平均不超过 3 秒 |
| 加载 1000 个账户数据的第一页 | 平均不超过 3 秒 |

## 10. 审计要求

必须记录审计日志的操作：

- 登录、登出、登录失败。
- 用户创建、删除、角色变更、密码重置。
- 案件创建、修改、删除。
- 账户导入、批量导入、删除、关联案件、余额同步。
- 托管钱包生成任务导出。
- 托管钱包生成结果导入。
- 交易创建、哈希生成、签名任务导出、签名结果导入、广播、确认。
- 热备份、冷备份导出、备份恢复。
- 测试数据生成和清理。

审计日志不能包含明文私钥、助记词、keystore、密钥分片、密码。

## 14. 验收口径

在线端达到以下条件即可支撑测试大纲中的在线侧要求：

- 用户和角色管理可用。
- 案件可创建、查询、修改和删除。
- 收缴原始账户可导入、批量导入、按案件查询和分页展示。
- 离线系统生成的案件托管钱包地址可导入在线系统。
- 在线系统可构建归集和提取交易，并生成待签名哈希。
- 在线系统可导出签名任务包、导入签名结果、广播交易并查询状态。
- 在线系统不保存私钥、不保存安全芯片和分片映射。
- 审计日志可记录和查询关键操作。
- 在线数据库可热备份、冷备份和恢复。
- 可生成或导入 1000 个账户和 500 笔交易测试数据。
- 关键页面和操作平均响应时间不超过 3 秒。
