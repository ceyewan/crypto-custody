# dto 目录

`dto` 目录定义接口请求、响应以及跨层传递的数据结构。DTO 用于隔离 HTTP 入参、业务处理结构和数据库模型，避免直接把 `model` 暴露给接口调用方。

## 文件说明

- `user_dto.go`：用户注册、登录、资料、密码和管理员操作相关结构。
- `account_dto.go`：账户创建、导入、查询和展示相关结构。
- `transaction_dto.go`：交易准备、签名发送、交易查询和统计相关结构。

## 维护建议

- 新增或修改接口字段时，优先在 DTO 中定义清晰的请求和响应结构。
- 外部接口字段使用稳定的 `json` 标签，不直接依赖数据库字段名。
- 请求 DTO 应保留必要的校验标签，便于 handler 层统一绑定和校验。
