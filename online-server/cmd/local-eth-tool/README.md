# local-eth-tool

本工具用于本地以太坊测试链调试，适合配合 Ganache 验证账户生成、测试 ETH 转账和离线签名广播流程。

## 运行方式

```bash
cd online-server
go run ./cmd/local-eth-tool
```

默认 RPC 地址：

```text
http://127.0.0.1:7545
```

## 功能

1. 生成新的以太坊地址、公钥和私钥。
2. 使用转出方私钥直接签名并发送 ETH 转账。
3. 根据交易明细生成待签名 Hash，等待离线端或门限签名系统生成签名后，再粘贴签名并广播交易。
4. 输入私钥和待签名 Hash，输出标准以太坊签名，用于模拟离线签名结果。

第 3 种模式和在线服务端交易流程一致：先构造原始交易并生成待签名 Hash，再使用外部签名附加到交易，验证签名地址与转出方一致后广播。

第 4 种模式可以和第 3 种模式配合使用：先用第 3 种模式生成交易 Hash，再用第 4 种模式使用转出方私钥对 Hash 签名，最后把签名粘回第 3 种模式广播交易。

## 推荐测试流程

### 生成测试账户

选择功能 `1`，工具会输出：

- 地址：用于接收测试 ETH 或导入系统账户管理。
- 私钥：用于本地测试签名。
- 公钥：用于核对账户材料。

### 给测试账户转 ETH

选择功能 `2`，输入 Ganache 中已有余额账户的地址和私钥，再输入目标地址和金额。该模式会直接完成签名和广播，适合给新生成账户分配测试 ETH。

### 模拟在线/离线签名链路

1. 选择功能 `3`，输入转出地址、转入地址和转账金额。
2. 工具输出交易明细和 `Message Hash`。
3. 暂时不要结束流程，保留等待输入签名的窗口。
4. 新开一个终端，选择功能 `4`，输入转出方私钥和上一步生成的 `Message Hash`。
5. 将功能 `4` 输出的签名复制回功能 `3`。
6. 工具会验证签名对应地址是否等于转出方地址，验证通过后广播交易。

该流程用于模拟在线系统生成待签名交易、离线系统完成门限签名、在线系统接收签名并广播交易的完整链路。

## 注意事项

- Ganache Desktop 中 `Network ID` 和 `Chain ID` 可能不同，交易签名必须使用 `eth_chainId`。本工具会自动读取并打印两者。
- 输入的外部签名应为 65 字节十六进制，格式为 `r(32) + s(32) + v(1)`。
- 签名的 `v` 值可以是 `0/1` 或 `27/28`，工具会自动适配。
- 只在本地测试链使用测试私钥，不要输入真实资产账户私钥。
