# tests 目录

`tests` 目录保存接口测试、集成测试和少量辅助验证程序。

## 子目录说明

- `user_test/`：用户注册、登录、令牌、资料、密码和管理员用户管理测试。
- `account_test/`：账户查询、创建、导入和管理员账户管理测试。
- `transaction_test/`：余额查询、交易准备、签名发送和交易查询测试。
- `web3/`：以太坊交互相关的辅助验证程序。

根目录下的 `user_test.go` 是独立的用户模块测试，使用测试环境配置和内存级辅助逻辑。

## 运行建议

普通 Go 测试：

```bash
go test ./...
```

接口测试需要先启动本地服务：

```bash
go run .
```

然后进入对应测试目录执行 `go test -v`。

## 注意事项

- 接口测试会修改本地数据库数据。
- 交易测试可能广播测试网交易，只能使用测试网络账户和测试资产。
- 管理员和警员相关测试需要提前准备对应账号和环境变量。
